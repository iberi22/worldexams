# Question
In double precision floating point, how many bits are used for the significand?

# Options
- [ ] A) 23
- [ ] B) 32
- [ ] C) 48
- [x] D) 52

# Explanation
Double precision floating point (64-bit) uses 52 bits for the significand (mantissa), which is more than double the 23 bits used in single precision. The complete double precision format is: 1 sign bit + 11 exponent bits + 52 significand bits = 64 bits total.
- A) 23 bits is the significand size for single precision (32-bit)
- B) 32 is the total size of single precision, not the significand of double
- C) 48 is not used in standard floating point formats
