# Contributing to World Exams

This project adheres to the **Git-Core Protocol** standards.

## Directory Structure

- `docs/specs`: Technical specifications and protocols.
- `docs/reports`: Analysis and progress reports.
- `docs/archive`: Deprecated or superseded documents.
- `src/`: Source code.
- `api/`: Static API data.

## Workflow

1. **Issues**: All work must start with a GitHub Issue or Task.
2. **Branches**: Use descriptive branch names (e.g., `feat/cloudflare-deploy`).
3. **Commits**: Use Conventional Commits (`feat:`, `fix:`, `docs:`).
4. **PRs**: Link to issues and provide comprehensive descriptions.
